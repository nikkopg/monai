# Milestone v1.0: Agentic Chat + Investments + Multi-page UI + MCP

**Status:** ✅ SHIPPED 2026-07-17
**Phases:** 1-7
**Total Plans:** 30

## Overview

monai became a real multi-page agentic personal-finance app. This milestone turned
a single-page NL-to-SQL prototype into a four-page app (chat, cashflow, investments,
settings) with an **agentic** chat that can both read and *safely edit* financial
data behind a confirm-before-write approval gate, plus an MCP server exposing the
read tools to external clients. It added Alembic-managed schema, API-key auth,
Decimal money math, a live-priced investment subsystem with multi-platform /
multi-currency / cash / gold positions, and allocation + historical-value charts.

## Milestone Summary

**Sequencing note:** Phase 7 (Investment Subsystem v2) was executed ahead of Phase 6 (MCP Server) — it completed 2026-07-13 after real dogfooding of Phase 5 surfaced multi-platform/currency gaps, while Phase 6 completed 2026-07-15.

**Key Decisions:**

- Agent reasons + chains *safe parameterized tools* only (no free SQL) — preserves correctness-by-construction from the tool-router pivot. ✓ Good.
- Agent writes use confirm-before-applying (UUID+TTL proposal, single-use token) + audit log — never mutate user data without explicit approval. ✓ Good.
- One MCP server (FastMCP co-mounted in FastAPI) powers web chat + external clients; external clients get read tools only. ✓ Good.
- Holdings identity moved from global `ticker` to `(ticker, platform_id)` for multi-platform positions. ✓ Good (Phase 7).
- Native-currency cost basis + FX conversion at display, with an immutable fx_rate_cache. ✓ Good (Phase 7).
- Cash and physical gold modeled as first-class asset types. ✓ Good (Phase 7).

**Issues Resolved:**

- Confident-wrong-number risk contained by keeping the LLM off raw SQL (tool registry only).
- Phase 2 human-verify surfaced 4 SSE/ProposalCard defects — all fixed as quick tasks.
- July-11 corrupt BTC price data (billion-USD price_cache entries, IDR/USD conflation, orphan NULL-platform holdings) corrected during Phase 7.
- find_platforms/find_accounts were omitted from the agent tool layer — wired in (07-05).

**Technical Debt Incurred (non-blocking):**

- Suggested test `test_mcp_trailing_slash_requires_key` for the `/mcp/` trailing-slash auth boundary (from 06-SECURITY.md).
- `backend/main.py` `_execute_proposal_payload` delete_holding branch inlines AuditLog instead of calling `apply_delete_holding` (writes.py) — drift risk, not a bug.
- Human-verify visual-only items (backend contracts verified programmatically): Phase 2 streaming ProposalCard render, Phase 5 INV-05 staleness badge pixel render, Phase 5/7 CHAT-03 live-LLM tool selection (non-deterministic).
