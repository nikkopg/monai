<objective>
Harden MONAI_API_KEY misconfiguration handling so a missing/empty key fails loudly instead of costing an hour of live debugging.

Two independent failure surfaces, both fixed:
1. `docker compose up`/`config` silently interpolates an unset `${MONAI_API_KEY}` to empty string, starting containers with no key.
2. With an empty `_CONFIGURED_KEY`, every write request hits the fail-closed guard which raises an *unhandled* `RuntimeError` → opaque `text/plain` 500 with no detail.

Purpose: Turn both silent misconfigurations into fast, actionable errors while staying fail-closed (no silent open writes).
Output: Updated compose required-var syntax, a 503 JSON auth guard, and test coverage for the empty-key path.
</objective>
