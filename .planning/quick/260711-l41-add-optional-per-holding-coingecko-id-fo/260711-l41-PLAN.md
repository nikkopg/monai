# Quick 260711-l41 — Per-holding `coingecko_id` (Tier 1)

## Context / decisions (locked)

- **Route (confirmed):** `POST /holdings` (`HoldingCreate`) and `PUT /holdings/{id}`
  (`HoldingUpdate`, serialized `model_dump(exclude_none=True)`) both flow through
  `apply_add_holding` / `apply_edit_holding`. **HoldingOverrideModal** hits these →
  the input belongs here. **HoldingModal** ("Log event") hits `/api/portfolio-events`
  (`PortfolioEventCreate`), which does NOT accept `coingecko_id` → **do not add the
  input there** (we just removed a dead Notes input for exactly this reason).
- **Set-when-provided** (None = keep) — mirror the existing `platform_id`/`asset_type`
  handling in `apply_edit_holding`. `PUT` already sends `exclude_none=True`, and the
  helper also guards `is not None`; keep both.
- **No fabrication:** an explicit `coingecko_id` is used verbatim as the CoinGecko id;
  a blank/omitted id falls back to the fixed symbol map (unknown → None → stale, never
  a guessed coin).
- Current Alembic head = `b2e6d4a19f73` (004_investment_platforms) → new revision's
  `down_revision`.
- Baseline: **137** backend tests green.

## Task 2 — Price adapter routing (TDD)

**Files:** `backend/prices.py`, `backend/tests/test_prices.py`

- `fetch_crypto_price(ticker: str, coin_id: str | None = None)`:
  - `resolved = coin_id if coin_id else TICKER_TO_COINGECKO_ID.get(ticker.upper())`
  - `if resolved is None: return None` — then use `resolved` as the CoinGecko `ids` param.
    (Keep the existing httpx call, error handling, and `(Decimal, "coingecko")` return.)
- `refresh_all_prices`: the call site currently does `adapter(h.ticker)` uniformly. Smallest
  correct change: when the routed adapter is the crypto one (i.e. `h.asset_type == "crypto"`,
  or `adapter is fetch_crypto_price`), call `fetch_crypto_price(h.ticker, h.coingecko_id)`;
  all other adapters stay `adapter(h.ticker)`. **Do not** change the `(ticker)` signature of
  `fetch_idx_price` / `fetch_manual_price`.

**Tests (extend test_prices.py, mock httpx as existing tests do):**
- explicit `coin_id="bittensor"` for an unknown ticker → uses `bittensor` (ignores the map), returns a Decimal.
- `coin_id=None`, ticker `"BTC"` → falls back to map (`bitcoin`) — regression guard.
- explicit `coin_id` empty string / None with unknown ticker → None (no fabricated call).

**Done:** targeted `test_prices.py` green in the rebuilt container.
