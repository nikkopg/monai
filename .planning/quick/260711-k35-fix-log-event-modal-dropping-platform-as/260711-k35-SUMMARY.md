# Quick Task 260711-k35: Fix Log-event modal dropping Platform/Asset type Summary

**Wired platform_id + asset_type through PortfolioEventCreate -> apply_add_portfolio_event -> holding, with set-when-provided no-clobber semantics, and fixed HoldingModal to actually send them.**

## Performance

- **Duration:** 25 min
- **Started:** 2026-07-11T07:15:00Z
- **Completed:** 2026-07-11T07:40:00Z
- **Tasks:** 2 completed
- **Files modified:** 4

## Accomplishments
- A buy event carrying `platform_id`/`asset_type` now lands those values on the recomputed holding (previously always Unassigned/null regardless of what was selected in the modal).
- A subsequent event on the same ticker with those fields omitted no longer clobbers the existing assignment back to null — matches the existing `apply_edit_holding` None-means-keep convention.
- `HoldingModal.handleSubmit` now includes `asset_type` and `platform_id` (parsed int, or `null` for "(unassigned)") in the POST body — the Platform/Asset type `<select>` inputs were already rendered but their values were silently dropped before this fix.
- Removed the dead Notes input and its state from HoldingModal (backend has no notes column; adding one is a migration, out of scope for this fix).

## Task Commits

Each task was committed atomically:

1. **Task 1: Add optional platform_id/asset_type to PortfolioEventCreate and set them on the holding in apply_add_portfolio_event** - `5af705e` (feat)
2. **Task 2: Wire platform_id + asset_type into HoldingModal.handleSubmit and remove the dead Notes input** - `3734ff5` (fix)

_Note: Task 1 carried `tdd="true"`; the new test and implementation landed together in a single commit rather than as separate RED/GREEN commits — see TDD Gate Compliance note below._

## Files Created/Modified
- `backend/schemas.py` - `PortfolioEventCreate` gains `platform_id: int | None = None` and `asset_type: str | None = None`, mirroring `HoldingCreate`
- `backend/writes.py` - `apply_add_portfolio_event` sets `holding.platform_id`/`holding.asset_type` after `recompute_holding_from_events`, only when the payload provides a non-None value (set-when-provided)
- `backend/tests/test_write_tools.py` - new test `test_apply_add_portfolio_event_sets_platform_and_asset_type_without_clobber` covers both the "sets on first buy" and "not clobbered by a later event with fields omitted" cases
- `ui/app/investments/HoldingModal.tsx` - `handleSubmit` POST body now includes `asset_type`/`platform_id`; dead Notes `<input>` and its `useState` removed

## Decisions Made
- Notes dropped from scope entirely (not persisted, not sent) — `PortfolioEvent` has no notes column in `backend/models.py`, and adding one needs a migration story the project doesn't have yet (no Alembic). Matches the plan's explicit "Notes decision" in the objective.
- `asset_type` kept as a free string field (not constrained to a `Literal`) on `PortfolioEventCreate`, consistent with `HoldingCreate`'s existing style — avoids coupling the backend schema to the frontend's `ASSET_TYPES` tuple.

## Orchestrator Addendum (post-merge) — 2026-07-11

The executor's "136 passed" was a **false green**: the `backend` service has no
source volume mount (`docker-compose.yml`), so `docker compose exec backend pytest`
ran the stale baked-in image, silently skipping the new test. After merging and
rebuilding the image (`docker compose up -d --build backend`), collection went
137 and the new test **failed**: `platform_id` came back `None`.

Root cause: `recompute_holding_from_events` adds the new `Holding` without
flushing, and the session is `autoflush=False` (`backend/db.py:17`), so the
set-when-provided lookup in `apply_add_portfolio_event` queried before the row
existed and set nothing on a first buy event.

Fix (commit `0c6c3b2`): `db.flush()` before the holding lookup in
`apply_add_portfolio_event`.

Verified on the rebuilt image: **137/137 passed**, the previously-failing test
green, `tsc --noEmit` clean. **Live browser UAT (closes deferred D4):** logged a
buy (ETH, crypto, platform Pintu) via the real `/investments` modal → holding
landed under **Pintu / asset_type crypto**; Notes input confirmed removed.
Lesson reinforced: rebuild the image before trusting container test runs
(see project memory "Deploy requires rebuild").
