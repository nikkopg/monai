## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| scheduler job → DB | A background job runs with implicit trust (no request auth) against price_cache + portfolio_value_history |
| external price APIs → daily job | The job makes the same outbound CoinGecko/yfinance calls as request handlers |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05-06-DOS | Denial of Service | AsyncIOScheduler daily job | high | mitigate | `max_instances=1` prevents overlapping runs exhausting DB connections; `misfire_grace_time=3600` + `coalesce=True` avoid a thundering catch-up after downtime (Pitfall 3); the job reuses the same `get_session_sync()` path as request handlers — no new trust boundary or elevated session |
| T-05-06-DEG | Denial of Service | daily_portfolio_snapshot_job | medium | mitigate | Per-ticker try/except inside refresh + snapshot so one failing external source never aborts the whole run or the scheduler thread (unit-tested in Task 1) |
| T-05-06-INT | Tampering | portfolio_value_history rows | low | accept | Duplicate same-day rows prevented by the unique (snapshot_date, ticker) index (Plan 01); a missed day is acceptable and non-backfillable by design (D-13) |
</threat_model>
