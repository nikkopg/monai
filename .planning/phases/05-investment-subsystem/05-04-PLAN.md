<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user ticker → outbound CoinGecko/yfinance HTTP | A user-controlled ticker string could steer an outbound request (SSRF) |
| browser → /prices/override, /prices/refresh | Untrusted price input / refresh trigger crosses into price_cache |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05-04-SSRF | Tampering / Information Disclosure | fetch_crypto_price / fetch_idx_price | high | mitigate | Crypto ticker resolved only via the fixed server-side `TICKER_TO_COINGECKO_ID` map (unknown → None), never interpolated as a free-text URL param; IDX ticker validated against stored `holdings.ticker` (alphanumeric String(32)) before `yfinance.Ticker(f"{ticker}.JK")` — no arbitrary outbound URL is user-constructible |
| T-05-04-AC | Elevation of Privilege | POST /prices/override, /prices/refresh | high | mitigate | Both write routes carry `dependencies=[Depends(require_api_key)]` (V4) |
| T-05-04-INP | Tampering | PriceOverrideRequest | medium | mitigate | price validated as positive Decimal at the schema boundary (V5); ticker validated against stored holdings |
| T-05-04-DEG | Denial of Service | refresh_all_prices | medium | mitigate | Per-ticker try/except (adapters return None, never raise) so one failing/slow external source never 500s the endpoint or aborts the batch (Pitfall 2); httpx timeout bounds each call |
| T-05-04-REP | Repudiation | apply_set_price | medium | mitigate | Manual price override writes an AuditLog row via writes.py (D-16) |
</threat_model>
