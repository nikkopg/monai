<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → Next.js proxy → API | Read-only fetch of /investments/history (open read) |
| client → GET /investments/history | range query parameter crosses here (untrusted input) |
| API → PostgreSQL | Read-only aggregation over portfolio_value_history; parameterized SQL only |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-07-04-INJ | Tampering | range query parameter | medium | mitigate | Validate range against a known set of tokens; unrecognized → ValueError→422; never interpolate the raw param into SQL (parameterized bound params only, project convention) |
| T-07-04-INT | Information Disclosure | P&L series correctness | medium | mitigate | P&L = market_value − cost_basis read directly from the already-audited snapshots (D-13) — no client-side money math; Decimal summation server-side |
| T-07-04-DoS | Denial of Service | unbounded history query | low | mitigate | Range filter caps the row scan; single-user app, portfolio_value_history is small and indexed on snapshot_date |
| T-07-04-XSS | Tampering | Recharts rendering | low | mitigate | Numeric IDR values from the trusted endpoint rendered via Recharts/React auto-escaping; no dangerouslySetInnerHTML (clone TrendChart's safe rendering) |
| T-07-04-SC | Tampering | npm installs | low | accept | Recharts ^3.9.2 already installed; no new package |
</threat_model>
