`backend/fx.py` adapter registry (structural clone of `backend/prices.py`, D-08), an

valuation (Plan 02), and historical-at-purchase P&L (Plan 02) all call `fx.get_rate()`.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| backend → frankfurter.dev | Outbound HTTP to a third-party FX vendor; `currency` values (FX-02 arbitrary) cross into an external URL |
| app → PostgreSQL | fx_rate_cache writes; parameterized SQL only |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-07-01-SSRF | Tampering | fetch_frankfurter_rate URL build | high | mitigate | Validate base/quote against `^[A-Z]{3,4}$` BEFORE URL interpolation (Pitfall 5); never pass a raw currency string to the external host — asserted in test_fx.py via mock.assert_not_called |
| T-07-01-DEG | Denial of Service | frankfurter.dev outage / slow response | high | mitigate | Adapter never raises (returns None on any failure) + short httpx timeout (mirror prices.py) so a vendor outage can never 500 or hang the endpoint; cached dates keep serving |
| T-07-01-INJ | Tampering | frankfurter JSON response parsing | medium | mitigate | Parse only rates[quote] as Decimal(str(...)) inside try/except; malformed/unexpected JSON returns None — no eval, no trust of response shape |
| T-07-01-INT | Repudiation | FX-converted money integrity | high | mitigate | All rates Decimal(str(x)), never float; adapter None propagated (no rate=1.0 fallback) so a confident-wrong number is never produced |
| T-07-01-IMM | Tampering | fx_rate_cache row overwrite | medium | mitigate | UniqueConstraint uq_fx_rate_cache_date_pair + get_rate only INSERTs on miss (never UPDATE) — a stored historical rate is immutable (FX-05), keeping historical P&L reproducible |
| T-07-01-SC | Tampering | npm/pip/cargo installs | low | accept | No new packages this phase (RESEARCH Package Legitimacy Audit: httpx already present); nothing to install |
</threat_model>
