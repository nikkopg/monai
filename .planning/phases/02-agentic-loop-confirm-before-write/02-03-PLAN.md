<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → /query-stream | Untrusted question crosses into the streaming agent (public read path) |
| browser → proxy → confirm/reject | API key injected server-side by the Next.js proxy; client never holds MONAI_API_KEY |
| SSE answer event → browser | Carries proposal_id (+ single-use confirm token for the just-created proposal) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-12 | Information Disclosure | confirm token in browser | accept | Token is single-use + 15-min TTL; surfaced only for the just-created proposal, never persisted; GET /proposals still excludes it (D-09/D-11) |
| T-02-13 | Spoofing | confirm/reject via proxy | mitigate | MONAI_API_KEY injected server-side in the proxy; browser never sees or sends it directly |
| T-02-14 | Denial of Service | SSE stream | mitigate | Agent `max_iterations=10` + `timeout=120.0` (Plan 01) bound stream duration; proxy passes through, no extra buffering |
| T-02-15 | Tampering | stale-tab approve | mitigate | Client greys expired cards (D-10) AND server enforces expiry → 410 (Plan 02); UI is cosmetic, server is authoritative |
| T-02-SC | Tampering | npm/pip installs | accept | No new packages — existing React 18 / Next 14 handle fetch+ReadableStream natively (RESEARCH) |
</threat_model>
