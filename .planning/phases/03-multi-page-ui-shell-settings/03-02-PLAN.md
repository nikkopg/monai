the live Postgres.

## Phase Goal (backend slice)

**As a** self-hosted single-user monai owner, **I want to** save my LLM provider,
model, API keys, base currency, and price data source to the backend, **so that** the
app uses them on the next request and remembers them across browser sessions and
container restarts without me editing env files.

> Derived from the ROADMAP Phase 3 `**Goal:**` line and success criteria 3-4. This plan
> delivers the persistence + reconfiguration engine; plan 03-03 delivers the browser form.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → PUT /settings | Untrusted, state-changing settings write (provider, keys, preferences) crosses here |
| GET /settings → client | Sensitive stored API keys must never cross back in raw form |
| app_settings (DB) → configure_llm | Stored raw keys read internally to configure the LLM client |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-10 | Information Disclosure | GET /settings response | high | mitigate | SettingsOut has no raw key field; get_effective_settings(raw_keys=False) returns only mask_key output; test_key_is_masked_never_raw asserts the raw key string is absent from the GET body. |
| T-03-11 | Tampering | PUT /settings (unauthenticated) | high | mitigate | PUT gated by existing require_api_key (hmac.compare_digest, fail-closed 503, 401 on mismatch); test_put_settings_requires_key asserts 401 without the header. |
| T-03-12 | Tampering / Availability | upsert_settings blank-string key overwrite | medium | mitigate | upsert_settings skips None and empty-string values (keep-existing); enforced server-side, not just via UI hint; test_blank_key_keeps_existing asserts a stored key survives a blank PUT. |
| T-03-13 | Repudiation / Trust | Stale LLM singleton after provider switch | medium | mitigate | reset_engine() called synchronously within the PUT after commit whenever an LLM field changed; test_llm_change_resets_engine asserts exactly-once on LLM change, never on preferences-only. |
| T-03-14 | Information Disclosure | AuditLog after-payload for settings | high | mitigate | PUT audit row stores key values only after passing them through mask_key; raw keys never persisted to audit_log. |
| T-03-15 | Repudiation | Cryptography at rest (plaintext keys in JSONB) | low | accept | Consistent with existing plaintext MONAI_API_KEY env var and CLAUDE.md single-user/self-hosted trust model (RESEARCH.md Assumption A1); no encryption-at-rest introduced this phase. |
</threat_model>
