## Spacing Scale

Declared values, extracted from the shipped `card`/`input`/`btn` constants and `main` page padding in `ui/app/page.tsx` (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps (e.g. `›` step marker), checkbox right-margin, label bottom-margin |
| sm | 8px | Button row gap, input-to-button gap, nav link internal gap |
| md | 16px | Form grid gaps (`gridTemplateColumns` rows), nav-link-to-nav-link spacing |
| lg | 20px | Card padding (`card.padding`), card bottom-margin (`card.marginBottom`) — preserved as-is from existing `card` constant, do not round to 24 |
| xl | 28px | Page title bottom-margin (existing `h1` block spacing) |
| 2xl | 40px | Page outer padding (`main` horizontal/vertical padding) |

Exceptions: the existing `input` (`padding: "10px 12px"`) and `btn` (`padding: "10px 18px"`) constants use 10px/12px/18px, which are pre-existing shipped values being extracted into `ui/app/styles.ts` unchanged — not redesigned this phase. New Phase 3 elements (Nav bar, Settings page sections) must use only the token scale above; do not introduce further off-scale values.

**Nav-specific:** horizontal nav bar height 56px (multiple of 4), 24px horizontal outer padding, 24px gap between the four nav links, sticky at top (`position: sticky; top: 0`) with the card border color as its bottom border (see Color).

## Typography

Consolidated from the sizes already in use (11/12/13/14/15/28px) down to the 4 declared here. New Phase 3 UI (Nav, Settings) must use only these — existing components already shipped (e.g. `ProposalCard`'s 11px uppercase micro-label) are left untouched since Phase 3 does not modify them.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Label | 12px | 400 | 1.4 |
| Body | 14px | 400 (600 for button/CTA text only) | 1.5 |
| Heading | 20px | 600 | 1.2 |
| Display | 28px | 600 | 1.2 |

Weights: exactly 2 declared — **regular (400)** for body copy, inputs, nav links, table/status text; **semibold (600)** for buttons/CTAs, the nav brand wordmark, active nav link, and page-level `Display`/`Heading` text. (Existing `btn` constant already uses 600 — reused, not new.)

Usage map for Phase 3 elements:
- Nav brand ("monai") — Heading, 20px/600, color `#e6e8eb`. Replaces the old page-level `<h1 style={{fontSize:28}}>` now that it lives in a persistent nav bar; keeps the wordmark readable at nav-bar scale.
- Nav links (Chat / Cashflow / Investments / Settings) — Body size (14px), weight 400 inactive / 600 active, per Color contract below for the color swap.
- Page-level heading on `/settings` and `/investments` (e.g. "Settings", "Investments") — Display, 28px/600.
- Settings card section titles ("LLM Provider & Model", "API Keys", "Preferences") — Label style (12px/400, color `#9aa0a6`), matching the existing established pattern where every `card` section is headed by the `label` constant (e.g. "Log a transaction", "Recent transactions" in current `page.tsx`). Do not promote these to Heading size — consistency with existing cards takes precedence.
- Form field labels, inline status/error text, masked-key hints — Label, 12px/400.
- Input values, button labels, table cells — Body, 14px.

## Color

60/30/10 split extracted from `ui/app/layout.tsx` + `ui/app/page.tsx` (dark palette, unchanged — no new palette invented):

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0f1115` | Page/body background, input field background |
| Secondary (30%) | `#1a1d23` | Card surfaces, nav bar background |
| Accent (10%) | `#3b82f6` | Reserved for: primary CTA buttons (Save buttons on Settings, Ask button), active nav link text + active-link underline/indicator, focus outline on inputs |
| Destructive | `#f87171` | Reserved for: Reject button (existing), negative/expense amounts, error/failure inline text (e.g. "Save failed") |

Supporting neutrals already established (not new, reused for Phase 3 elements):

| Token | Value | Usage |
|-------|-------|-------|
| Border | `#2a2e37` | Card borders, table row dividers, nav bar bottom border, input borders |
| Text primary | `#e6e8eb` | Body text, input values, nav brand |
| Text muted | `#9aa0a6` | Labels, section titles, inactive nav links, placeholder-style hints |
| Success | `#4ade80` | Reserved for: Settings save-success inline message, existing Approve button/applied banner (unchanged) |
| Disabled | `#374151` | Disabled button background (Save buttons while pending) |

Accent reserved for: active nav link (text color + 2px bottom indicator bar), and the three Settings "Save" buttons (Save Provider, Save Keys, Save Preferences). Accent is not used for card borders, body text, or decorative elements — matches existing restraint in `page.tsx` (`#3b82f6` only appears on the primary "Ask" button, the `›` step marker, tool names in the trace, and the pending-proposal card border).

## Layout & Component Notes (supplemental — for planner/executor)

These are not part of the template's fixed sections but are load-bearing for implementation; included so the plan doesn't have to re-derive them.

**`<Nav/>` component** (`ui/app/components/Nav.tsx` or similar, rendered once in `ui/app/layout.tsx`):
- Client component (`"use client"`) — uses `usePathname()` from `next/navigation` for active-link detection.
- Layout: horizontal flex row, brand wordmark left-aligned, four nav links right-aligned (or immediately following brand — researcher default: brand left, links follow left-to-right in requirement order Chat/Cashflow/Investments/Settings, no right-alignment needed for a 4-link nav).
- Each link is a `next/link` `<Link>` (client-side transitions, satisfies UI-02's "no full page reload").
- Active state: text color Accent (`#3b82f6`) + weight 600 + 2px bottom border in Accent. Inactive: text muted (`#9aa0a6`) + weight 400, no border.
- Sticky top, `zIndex` above page content, bottom border `1px solid #2a2e37`, background Secondary (`#1a1d23`) to visually separate from the page's Dominant background.

**Page skeletons:**
- `/chat` — existing ask box + SSE streaming + ProposalCard + tool trace section from current `page.tsx`, moved intact (no visual changes).
- `/cashflow` — existing manual-entry form + recent-transactions table sections from current `page.tsx`, moved intact as interim content (full dashboard is Phase 4).
- `/investments` — new skeleton: page Display heading "Investments are coming in Phase 5" inside a single `card`, with the empty-state body copy beneath it. No table, no form — this is intentionally sparse per CONTEXT.md ("real skeleton page," not a stub route).
- `/settings` — new page: Display heading "Settings", then three `card` sections in order: "LLM Provider & Model" (provider dropdown + model text input + Save Provider button), "API Keys" (masked password-type inputs per configured provider + Save Keys button), "Preferences" (base currency input/select + price data source dropdown + Save Preferences button). Each section has its own inline success/error message per the Copywriting Contract.
- Root `/` — redirect to `/chat` (server-side `redirect()` in `app/page.tsx`, per CONTEXT.md discretion note; a server-component redirect is simpler than a `next.config.js` rewrite and keeps routing logic colocated with the App Router).

**Settings form field types (from CONTEXT.md, locked):**
- Provider: `<select>` styled with the `input` constant — options `ollama` / `claude` / `openai`.
- Model: `<input type="text">` — pre-filled with the provider's current/default model on load and on provider change.
- API keys: `<input type="password">` — placeholder shows masked value from server; empty value on submit means "keep existing."
- Base currency: `<input type="text">` (default `IDR`) — free text is acceptable since currency validation is out of scope this phase (single-currency assumption holds per CLAUDE.md).
- Price data source: `<select>` — options `coingecko` / `yfinance` / `manual`.
- All Save buttons: reuse `btn` constant; disabled + `"Saving…"` label while a request is pending (existing pattern from `addTx`/`ask` in `page.tsx`).
