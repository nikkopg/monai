<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser form → Next proxy → write endpoints | User-entered transaction/account/category data and the uploaded CSV cross into the DB; the proxy injects the API key server-side |
| account delete UI → reassign-then-delete endpoint | The UI drives the D-06 destructive flow; a wrong target could misroute transactions |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-16 | Elevation of Privilege | UI write calls | high | mitigate | All writes go through `/api/[...proxy]`, which injects MONAI_API_KEY server-side (existing, unchanged); the browser never holds the key; backend endpoints enforce require_api_key (Plan 03) |
| T-04-17 | Tampering / Repudiation | destructive delete without confirmation | medium | mitigate | ConfirmDialog gates every destructive action (delete tx/account, merge category — D-03); account delete additionally requires an explicit reassignment choice (D-06) so no silent orphaning/data loss |
| T-04-18 | Input Validation | CSV upload abuse (malformed rows, size) | medium | mitigate | Reuses the existing tested `POST /import` (backend parse_csv validates rows/currency and returns skipped counts; reads only the uploaded bytes, never a filesystem path — RESEARCH Security Domain); UI surfaces skipped count so malformed rows are visible, not silent |
| T-04-19 | Information Disclosure | error details shown in UI | low | accept | Error detail strings come from the app's own 422/500 responses (single-user local app); no cross-tenant leakage risk |
| T-04-SC | Tampering | npm/pip installs | high | mitigate | This plan installs NO new packages (recharts already added in Plan 04); no [ASSUMED]/[SUS] packages introduced |
</threat_model>
