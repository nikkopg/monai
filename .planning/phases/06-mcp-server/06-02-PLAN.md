<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| external MCP client → /mcp transport | Untrusted network client crosses into the co-mounted MCP app over financial data; gated by MONAI_API_KEY. |
| MCP tool call → backend/tools.py callables → PostgreSQL | Tool args cross into parameterized SQL; no free-form SQL is exposed. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-06-02-01 | Information Disclosure | unauthenticated /mcp access to finance data | high | mitigate | Outer-app middleware scoped to /mcp rejects missing/invalid key with 401 (503 if key unset) BEFORE the MCP session handles the request; asserted by `test_mcp_requires_key` (MCP-04). |
| T-06-02-02 | Elevation of Privilege / Tampering | write/propose_* tools reachable over MCP | high | mitigate | Only `backend.tools.TOOLS` (15 reads) are registered; the 11 `propose_*` callables are never passed to FastMCP → absent from tools/list. Asserted (absence, not block) by `test_mcp_no_write_tools` (D-03/MCP-03). |
| T-06-02-03 | Tampering | auth bypass via path handling (/mcp vs /mcp/ trailing slash, sub-paths) | high | mitigate | Guard matches `request.url.path.startswith("/mcp")`, covering the whole mounted subtree (/mcp, /mcp/, sub-paths); registered as outer-app middleware so it runs before the mounted sub-app (Pitfall 4). |
| T-06-02-04 | Information Disclosure | timing attack on API-key comparison | medium | mitigate | Reuse `hmac.compare_digest` via the single `auth.key_ok()` helper — no hand-rolled `==` (D-04 one check). |
| T-06-02-05 | Information Disclosure | MONAI_API_KEY leaked via logs/traces in the /mcp middleware | medium | mitigate | Middleware never logs the header value (mirrors existing auth silence); no new secret introduced (D-04 reuse). |
| T-06-02-06 | Tampering | free-form SQL injection via a "flexible" MCP tool | high | mitigate | No such tool exists — only the parameterized `TOOLS` callables are registered; do NOT add a raw-query tool (correctness-by-construction). Acceptance grep confirms no `text(`/raw SQL in mcp_server.py. |
</threat_model>
